## Описание алгоритма

Для каждого байта входного потока алгоритм анализирует предшествующие N байт
(контекст порядка N) и оценивает вероятности появления следующего символа. Полученные вероятности подаются
на арифметический (range) кодер, который кодирует символы практически оптимально (бесконечно близко к энтропии при
устремлении размера исходного файла к бесконечности).

## Энкодер

### Процесс кодирования

Кодирование выполняется конвейером из двух этапов: контекстное моделирование (PPM) и арифметическое кодирование.

#### 1. Контекстная модель PPM

Модель хранит дерево контекстов от порядка 0 до порядка N (задаётся параметром `modelOrder` в конструкторе).
Каждый узел дерева содержит статистику частот символов, встреченных после данного контекста.

Для каждого входного байта выполняется следующий процесс:

1. **Поиск символа в контекстах от максимального к минимальному порядку:**
    - Начиная с контекста порядка N (самого длинного), проверяется, встречался ли текущий символ в данном контексте
    - Если символ найден - кодируется его вероятность и процесс завершается
    - Если символ не найден - кодируется вероятность escape (перехода к более короткому контексту)
    - При escape все символы текущего контекста добавляются в множество исключений (encoding-time exclusion)
    - Переход к контексту порядка N−1 и повторение до порядка 0

2. **Порядок −1:**
    - Если символ не найден ни в одном контексте, он кодируется равномерным распределением
      среди оставшихся (не добавленных в excluded) символов из 256 возможных

3. **Обновление модели:**
    - После кодирования символа обновляются частоты во всех контекстах от порядка 0 до N
    - Путь контекстов сдвигается: для каждого порядка k новый контекст = старый контекст порядка k−1, расширенный
      текущим символом
    - При достижении суммарной частоты порога (задаётся параметром `rescaleThreshold` в конструкторе) выполняется
      рескейл - все частоты делятся пополам, нулевые записи удаляются

#### Метод D для оценки escape

Используется формула оценки вероятностей из метода D:

- **Вероятность символа s:** `P(s) = (2 * c(s) − 1) / 2N`, где `c(s)` - число наблюдений символа, `N` - суммарная
  частота
- **Вероятность escape:** `P(esc) = d / 2N`, где `d` - количество различных символов в контексте, `N` - суммарная
  частота

#### 2. Арифметическое кодирование

Range-кодер реализован по схеме Subbotin/Schindler с carry-propagation через кэш байтов:

- **Рабочий диапазон:** 32-битное целое `_range`, начальное значение `0xFFFFFFFF`
- **Нижняя граница:** 64-битное целое `_low` для обработки переноса (carry)
- **Нормализация:** при сужении диапазона ниже `2^24` - сдвиг и запись байта в выходной поток
- **Кэширование carry:** буфер из `0xFF` байтов для корректной обработки цепных переносов

Для каждого символа кодер выполняет:

```
range = range / totalFreq
low  += cumFreq * range
range = freq * range

normalize()
```

Где `cumFreq` - кумулятивная частота символов с меньшим значением, `freq` - частота данного символа,
`totalFreq` - суммарная частота всех символов (включая escape).

### Формат архива

В выходной файл последовательно записываются:

1. Размер исходного файла (`int32`, 4 байта)
2. Порядок контекстной модели (`byte`, 1 байт)
3. Арифметически закодированные данные

Фиксированный overhead метаданных: **5 байт** на файл.

## Декодер

### Процесс декодирования

Декодирование выполняется в обратном порядке с идентичной контекстной моделью:

1. Из архива считываются метаданные: размер оригинала (4 байта) и порядок модели (1 байт)
2. Инициализируется range-декодер: считываются первые 5 байт арифметического потока в рабочий регистр `_code`
3. Для каждого байта оригинала:
    - Строится то же множество контекстов, что и при кодировании
    - По пороговому значению `threshold = code / range` определяется, какой символ (или escape) закодирован
    - При escape - спуск к контексту меньшего порядка с тем же механизмом encoding-time exclusion
    - После определения символа - обновление модели (идентично кодеру)
4. Восстановленные данные записываются в выходной файл

Корректность декодирования гарантируется тем, что кодер и декодер поддерживают идентичное состояние
контекстной модели на каждом шаге - обе стороны выполняют одинаковые обновления статистики после каждого символа.
